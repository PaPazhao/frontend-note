    事件

    # 事件绑定

    给元素绑定事件通常使用 addEventListener 方法, 而且同一个事件可以绑定多个监听者

    const btnEl = document.querySelector(".btn");
btnEl.addEventListener("click", () => {
  console.log("按钮 addEventListener click 事件 1");
});
btnEl.addEventListener("click", () => {
  console.log("按钮 addEventListener click 事件 2");
});

    也可以使用 onclick 来绑定事件,但是只能绑定一个事件(后绑定的事件会覆盖前面的)

    btnEl.onclick = function () {
  console.log("按钮 onclick 事件 1");
};
btnEl.onclick = function () {
  console.log("按钮 onclick 事件 2");
};

    onclick 绑定的事件和 addEventListener 绑定的事件可以同时触发

    # 常用的事件类型

    # 鼠标事件

    • click 鼠标点击
    • mouseover/mouseout 鼠标指针移入/离开一个元素
    • mousedown/mouseup 鼠标按下/释放按钮
    • mousemove 鼠标移动

    # 键盘事件

    • keyup/keydown 按下和松开一个案件时

    # 表单时间

    • submit 提交 <form> 时
    • focus 聚焦一个元素, 例如 input 输入时

    # Document 事件

    • DOMContentLoaded 当 HTML 的加载和处理都完成, DOM 完全构建时

    # CSS 事件

    • transitionend 一个 CSS 动画完成时

    # 事件冒泡和事件捕获

    事件冒泡就是当一个事件产生时, 这个事件会随着 DOM 链条从子节点依次向父节点传递的过程, 在冒泡的过程中如果冒泡到当前的节点给这个事件绑定了监听回调,那么回调会依次调用.

    const innerEl = document.querySelector(".inner-box");
const outerEl = document.querySelector(".outer-box");
innerEl.addEventListener("click", function () {
  console.log("inner-box click");
});

outerEl.addEventListener("click", function () {
  console.log("outer-box click");
});

document.body.onclick = function () {
  console.log("body click");
};

    事件捕获就是响应事件的方向跟冒泡相反, 事件捕获是从 document 开始

    冒泡和捕获其实本质没区别,只是历史原因. 不同的浏览器厂商对事件处理的顺序不同所以就是出现了冒泡和捕获.

    outerEl.addEventListener(
  "click",
  function () {
    console.log("outer-box click");
  },
  true // 默认值 false 冒泡, 向捕获传递 true
);

    一般默认情况就是事件冒泡.

    # 事件对象

    事件绑定的处理函数默认会传递一个 event 参数,这个就是事件对象. 它里面有很多属性

    # 事件对象常用的属性

    • type
    • target 事件发生的元素
    • currentTarget 当前处理事件的元素(就是这个回调函数绑定在哪个元素上)
    • eventPhase: 事件所处的阶段
      • 1: 事件捕获阶段
      • 2: 事件传递到发生点击事件的元素了
      • 3: 事件到冒泡的阶段

    当 target 和 currentTarget 相等的时候意味着事件发生的对象和事件处理的对象是同一个对象

    # 事件方法解析

    # 阻止默认行为

    例如点击一个 a 元素默认会跳转到对应的链接,如果想阻止这个默认跳转行为

    const aEl = document.querySelector("a");
aEl.addEventListener("click", function (event) {
  event.preventDefault();
});

    # 阻止事件传递

    有时候我们希望能够控制事件的传递过程,比如当前的元素处理了事件以后不希望事件继续传递,就可以使用 event.stopPropagation(); 方法

    const divEl = document.querySelector(".box");
divEl.addEventListener("click", function (event) {
  event.stopPropagation();
});

    # 事件处理中的 this

    事件处理函数中的 this 就是当前事件绑定的元素.也就是说给谁绑定的事件处理函数 this 就指向了谁

    # EventTarget 类

    所有的节点, 元素都继承自 EventTarget 类, 其实 window 也继承自 EventTarget.

    EventTarget 是一个 DOM 接口, 主要用来添加, 删除, 派发 Event 事件

    # EventTarget 常见的方法

    • addEventListener 注册事件以及事件处理函数
    • removeEventListener 删除事件类型和处理函数
    • dispatchEvent 派发事件到某个 EventTarget 对象

    # dispatch 事件

    有时候需要自定义事件,然后通过代码来触发

    // 派发自定义事件
const eEl = document.querySelector(".dispatch");
eEl.addEventListener("event-type", function (e) {
  console.log("自定义事件");
});
eEl.dispatchEvent(new Event("event-type"));

    # 事件代理

    一个元素有很多的子节点,例如一个 tabbar 里面有很多的 item. 我们可以把 item 的点击事件放到 tabbar 里面去统一处理, 这就是事件的代理.

    • 子元素点击时父元素可以通过冒泡传监听事件
    • 然后通过事件属性 event.target 来获取是哪个元素传递过来的事件
    <style>
  .active {
    color: red;
  }
</style>
<div class="delegate">
  <ul>
    <li>1</li>
    <li>2</li>
    <li>3</li>
    <li>4</li>
    <li>5</li>
  </ul>
</div>
<script>
  const ulEl = document.querySelector("ul");
  ulEl.addEventListener("click", function (event) {
    if (this.selectedItem) {
      this.selectedItem.classList.remove("active");
    }
    event.target.classList.add("active");
    this.selectedItem = event.target;
  });
</script>

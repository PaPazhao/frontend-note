    for-in 是如何遍历属性的, 他都能遍历哪些属性?

    当使用 for-in 遍历一个对象时他的原理和查找 [[Prototype]] 链类似, 任何可以通过原型链访问到的属性都可以被遍历到. 前提是属性是 enumerable

    例如, 定义两个对象 anotherObj 和 obj, 对象 obj 是以 anotherObj 为原型构建的, 使用 for-in 遍历对象 obj 的时候哪些属性可以被遍历到? for-in 遍历的过程是如何遍历的呢?

    // 1. 定义 anotherObj 对象
const anotherObj = {
  a: 1
}
Object.defineProperties(anotherObj, {
  'b': {
    enumerable: true,
    value: 2
  },
  'c': {
    enumerable: false,
    value: 3
  }
})

// 2. 定义 obj 对象
const obj = Object.create(anotherObj)
obj.d = 4
Object.defineProperties(obj, {
  'e': {
    enumerable: true,
    value: 5
  },
  'f': {
    enumerable: false,
    value: 6
  }
})

console.log(anotherObj)
console.log(obj)

for(const key in obj) {
  console.log(key)
}

// 结果:
// {a: 1, b: 2, c: 3} : anotherObj
// {d: 4, e: 5, f: 6} : obj
// d
// e
// b
// a

    # for-in 是如何遍历属性的

    对象都有一个特殊的内置属性 [[Prototype]] 他是对其他对象的引用

    var obj = {
  a: 2
}

obj.a // 2

    当引用对象的属性时会触发 [[Get]] 操作(比如上面代码 obj.a), 而 [[Get]] 操作会检查对象本身是否有这个属性. 如果没有找到那么需要使用对象的[[Prototype]]链了

    const anotherObject = {
  a: 2
}

// 创建一个关联到 anotherObject 的对象
const myObject = Object.create(anotherObject)
myObject.a // 2

    这里 myObject.a 的值为什么是 2?

    使用 Object.create 创建的对象 myObject 他的 [[Prototype]] 指向了 anotherObject. 当引用属性 a 的时候触发了 [[Get]] 会在当前对象上查找有没有属性 a ,显然是没有的,接着就会在 myObject 的[[Prototype]] 上查找,也就是 anotherObject 查找,找到了 a 的值 2. 假如anotherObject 没找到就会继续向上查找直到遍历整个原型链条. 遍历整个链条后依然没找到就返回了 undefined .

    for...in 遍历对象时原理和查找[[Prototype]]链类似,任何可以通过原型链访问到(并且是 enumerable)的属性都会枚举

    使用 in 操作符检查属性在对象中是否存在时会查找对象的整条原型链

    var anotherObject = {
  a: 2
}

// 创建一个关联到 anotherObject 的对象
var myObject = Object.create(anotherObject)
for(var k in myObject) {
  console.log('found:' + k)
}

// found: a
'a' in myObject // true

    因此当你通过各种语法进行属性查找时都会查找[[Prototype]] 链,直到找到属性或者查找完整条原型链

    # Object.prototype

    [[Prototype]] 的尽头是 Object.prototype . 因此 Object.prototype 包含了 JavaScript 中许多通用的功能.

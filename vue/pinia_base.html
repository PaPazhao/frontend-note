    Pinia 的基础使用

    # 安装

    npm install pinia

    # Pinia 的基本使用

    由于 vuex 在组合式 api 里各种难用, pinia 是用来替代 vuex 的库.

    # 创建 Store

    工程下创建目录文件: src/stores/index.js, 创建一个 pinia

    import { createPinia } from "pinia";

const pinia = createPinia()

export default pinia

    # 在 App 里引入

    上面创建好的 pinia 需要在创建的 App 里引入

    import { createApp } from 'vue'
import App from './views/App'
import pinia from './store'

createApp(App)
  .use(pinia)
  .mount('#app')

    这样 pinia 就可以在组件里面使用了. 下面以计数器案例来展示 pinia 如何使用

    # 定义 store

    和 vuex 的单一数据源不同, pinia 可以给每个组件设计一个单独数据源, 下面给计数器定义一个 store.

    定义 counter 的 store 文件: src/stores/counter.js

    import { defineStore } from "pinia";

const useCounter = defineStore('counter', {
  state() {
    return {
      count: 99
    }
  }
})

export default useCounter

    pinia 使用 defineStore 来定义 store, 他有俩参数

    • store 的名字: 'counter'
    • store 的内容, 他是一个对象有三个字段 state, actions, getters 这个跟 vuex 是一样的

    # 组件里使用 Store

    定义组件: counter.vue

    <template>
  <div class="counter"> 
    <h2>计数器</h2>
    <div>计数值: {{ counterStore.count }}</div> 
  </div>
</template>
 
<script setup>
import useCounter from "@/store/counter"; 
 
// 这里不能直接解构,会失去响应式 
const counterStore = useCounter()

</script>

    在组件里引入 counter 的 store. 这里就是调用 useCounter 函数返回了 counterStore 就可以使用了.

    # 组件里修改 store 的状态

    增加一个按钮,修改状态的值 count++ , 这里直接对 count 进行操作就行( counterStore.count++), 不需要像 vuex 那样 commit

    <template>
  <div class="counter"> 
    <h2>计数器</h2>
    <div>计数值: {{ counterStore.count }}</div> 
    <button @click="increment">计数值+1</button>
  </div>
</template>
 
<script setup>
import useCounter from "@/store/counter"; 
 
const counterStore = useCounter()
 
function increment() {
  counterStore.count++
}
</script>

    # 解构失去响应式的问题

    如果我们试图对 counterStore 进行解构 count 后使用, 那么 count 会失去响应式:

    const { count } = useCounter()

    解决办法

    1. 使用 toRefs()
    const counterStore = useCounter()
const { count } = toRefs(counterStore)
    1. 使用 pinia 提供的 storeToRefs()
    const counterStore = useCounter()
const { count } = storeToRefs(counterStore)

    # 直接读写 state

    通过 store 的实例读取,写入 state. 默认情况可以直接通过 store 来读写 state 的数据

    const userStore = useUser()
 
userStore.name = 'baby'
userStore.age = 100

    # $reset 重置 store

    使用 $reset 来重置 store 到初始状态

    const userStore = useUser()

// 1. 先修改 state
userStore.name = 'baby'
userStore.age = 100

// 2. 重置 userStore 到初始状态
userStore.$reset()

    # $patch 批量修改

    除了一个一个的修改 state 的属性,还可以使用 $patch 批量修改

    userStore.$patch({
  name: 'baby',
  age: 22
})

    # $state 整个替换 state

    userStore.$state({
  name: 'baby',
  age: 22 
})

    # getters

    getters 相当于 store 的计算属性,

    # 在 getters 定义一个 getter 属性

    可以用 defineStore() 中的 getters 属性定义 可以定义接收一个 state 作为参数的函数

    const useCounter = defineStore('counter', {
  state: () => ({
    count: 100
  }),
  getters: {
    doubleCount(state) {
      return state.count * 2
    }
  }
})

    组件里直接使用 counterStore.doubleCount 就可以访问了

    <template>
  <div class="counter"> 
    <div>计数值: {{ counterStore.doubleCount }}</div>  
  </div>
</template>

    # 一个 getter 使用另一个 getter

    定义一个 getter: doubleCountAddOne , 他内部调用 doubleCount

    const useCounter = defineStore('counter', {
  state: () => ({
    count: 100
  }),
  getters: {
    doubleCount(state) {
      return state.count * 2
    },

    doubleCountAddOne() {
      // 这里的 this 绑定了当前的 store 实例
      return this.doubleCount + 1
    }
  }
})

    # getter 支持返回函数

    例如 state 有个用户列表,然后通过 getUserInfo 返回某个用户的信息

    const useCounter = defineStore('counter', {
  state: () => ({
    userInfos: [
      {id: 1, name: '用户-1'},
      {id: 2, name: '用户-2'},
      {id: 3, name: '用户-3'},
    ]
  }),

  getters: {
    getUserInfo(state) => (id) => {
      state.userInfos.find(user => { 
        if(user.id === id) { 
          return user 
        }
      })
    } 
  }
})

    # getter 访问别的 store 的数据

    引入依赖的 store 然后使用就可以了

    const useUserInfo = defineStore('userInfo', {
  state: () => ({
    userInfos: [
      {id: 1, name: '用户-1'},
      {id: 2, name: '用户-2'},
      {id: 3, name: '用户-3'},
    ]
  }),

  getters: {
    showMessage(state) {
      // 1. 获取其他 store 的数据
      const userStore = useUser()

      // 2. 用自己的数据和别人的数据拼接
      return `${自己的数据} - ${别的数据}`
    }
  }
})

    # actions 的使用

    action 类似 vue 的 method. 可以直接操作 state 的数据, action 里面绑定了 this 可以直接使用.

    # action 同步方法

    const useCounter = defineStore('counter', {
  state: () => ({
    count: 100
  }),
 
  actions: {
    increment() {
      this.count++
    }
  }
})

    <template>
  <div class="counter">
    <h2>计数器 {{ counterStore.count }}</h2>
    <button @click="increment">+1</button>
  </div>
</template>
 
<script setup>  
import useCounter from '@/store/counter'
 
const counterStore = useCounter()

function increment() {
  counterStore.increment()
} 
</script>

    # action 的异步方法

    
import { defineStore } from "pinia";

const useHome = defineStore('home', {
  state: () => ({
    banners: []
  }),

  actions: {
    async fetchHomeData() {
      const res = await fetch('http://api.xx.com/homedata ')
      const data = await res.json()
 
      this.banners = data.data.banner.list
    }
  }
})

export default useHome

    vue3 的生命周期

    # 题目

    Vue 每个生命周期都做了什么?

    # Vue3 生命周期图示

    Vue 的声明周期从大的方面主要分下面几个阶段:

    • 创建: created
    • 挂载: mounted
    • 更新: updated
    • 卸载: unmounted

    在每个阶段 VUE 都提供了两个回调的钩子函数分别对应当前阶段开始前和当前阶段完成后:

    • beforeCreate

    • created

    • beforeMount

    • mounted

    • beforeUpdate

    • updated

    • beforeUnmount

    • unmounted

    • activated

    • deactivated

    # 生命周期钩子函数

    # 1.beforeCreate

    beforeCreate 被回调的时候仅仅是创建了一个 VUE 的实例,这个时候 data/method 等还没有初始化,所以在这个阶段你访问当前实例的 data 属性返回 undefined. 开发中这个阶段一般不会用到

    # 2.created

    created 回调的时候意味着 VUE 的实例已经初始化完成,尚未开始渲染模板.响应式/计算属性/方法/侦听器都已经起作用了.也就是说 VUE 的实例对象已经完全可用了.

    通常在这个阶段我们会做和 DOM 节点没有关系但是跟 JS 模型有关系的操作,比如网络请求数据/开启数据侦听/绑定事件等操作.

    # 3. beforeMount

    这个时期主要做的事情就是给 DOM 挂载做准备工作. 但是还不能访问 DOM

    • 模板解析编译
    • 把 VUE 的实例通过调用 render() 转换成虚拟节点 vdom
    • 还没有开始渲染 DOM

    这个钩子在服务端渲染时不会被调用.

    # 4. mounted

    这个时候组件已经挂载到 DOM 上了, 我们可以拿到 DOM 的元素了

    • 完成 DOM 渲染
    • 组件创建完成
    • 由创建阶段 进入 运行阶段
    • 同步子组件都已经被挂载(不包含异步组件或 <Suspense> 树内的组件)
    • 自身的 DOM 树已经创建完成并插入了父容器中. 注意仅当根容器在文档中才可以保证组件 DOM 树也在文档中.

    这个钩子通常用于执行需要访问组件所渲染的 DOM 树相关的副作用

    # 5. beforeUpdate

    当一个响应式状态改变引起组件更新 DOM 之前被调用

    • data 发生变化之后
    • 准备 更新 DOM (还没更新 DOM)

    这个钩子可以用来在 VUE 更新 DOM 之前访问 DOM 状态. 这个钩子中更改状态也是安全的.

    # 6. updated

    当一个响应式状态改变引起组件更新 DOM 之后被调用

    • data 发生变化之后
    • 更新 DOM 完成
    • 不要在 updated 里修改 data, 可能导致死循环

    父组件的更新钩子将在其子组件更新钩子之后调用

    这个钩子会在组件的任意 DOM 更新之后被调用,这些更新可能是由不同的状态变更导致的.如果你需要在某个特定的状态更改后访问更新后的 DOM,使用 nextTick()

    不要在 updated 钩子里更改组件的状态,这可能会导致无限更新循环

    # 7. beforeUnmount

    组件被卸载之前调用.这个钩子被调用时,组件实例依然还有全部功能

    • 组件进入销毁阶段(尚未销毁,可以正常使用)
    • 可以移除/解绑一些全局事件,自定义事件

    # 8. unmounted

    在一个组件实例被卸载后使用.

    • 组件被销毁了
    • 所有子组件也被销毁了

    一个组件在以下情况视为已经卸载:

    • 其所有子组件都被卸载
    • 所有相关的响应式作用(渲染作用以及 setup() 时创建的计算属性和侦听器)都已经停止

    可以在这个钩子里手动清理一些副作用, 例如计时器/DOM事件监听器或者服务器连接等

    # KeepAlive

    当使用 KeepAlive 缓存组件的时候会用到 activated 和 deactivated 两个钩子. 例如 tabControl 组件,当 tab 切换的时候需要卸载一个组件挂载另一个组件. 使用 KeepAlive 则可以把暂时不需要展示的组件缓存起来当需要展示的时候从缓存里拿出来展示,这样不仅仅可以优化性能而且还能保存组件之前的状态.

    # 9. activated

    当组件插入 DOM 中时候被调用.

    • 首次挂载
    • 从缓存中取出重新插入 DOM 的时候调用

    # 10. deactivated

    当组件从 DOM 中移除的时候被调用

    • 从 DOM 上移除进入缓存
    • 组件卸载
